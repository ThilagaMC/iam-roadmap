## 👩‍💻 About Me

I am an IAM professional with **3 years of hands-on experience** working with enterprise identity platforms including **Okta** and **Microsoft Entra ID (Azure AD)**. After building a strong foundation in day-to-day IAM operations, I decided to go deeper — expanding my knowledge across the full IAM domain including Identity Governance, Privileged Access Management, and IAM Architecture.

This repository documents my structured 10-week self-learning roadmap, built around concept-first learning with daily theory, exploration, and hands-on practice tasks.

## 📚 What I Am Learning

### ⚡ Phase 1 — Core IAM
- Authentication fundamentals: MFA, FIDO2, passwordless
- Federation protocols: SAML 2.0, OAuth 2.0, OpenID Connect (OIDC)
- Identity Lifecycle Management: Joiner / Mover / Leaver
- Single Sign-On (SSO) architecture and trust models
- Zero Trust Identity principles (NIST SP 800-207)
- Session management and token security

### 🌿 Phase 2 — Identity Governance & Administration (IGA)
- Access certification and review campaigns
- Role-Based Access Control (RBAC) design and role mining
- Segregation of Duties (SoD) and toxic combinations
- Automated provisioning via SCIM protocol
- Identity analytics, risk scoring, and UEBA concepts
- Entitlement management and access lifecycle
- IGA compliance frameworks: SOX, ISO 27001, HIPAA, NIST

### 🔐 Phase 3 — Privileged Access Management (PAM)
- Privileged account discovery and shadow admin detection
- Credential vaulting concepts and rotation policies
- Just-in-Time (JIT) access and standing privilege reduction
- Privileged session monitoring and recording
- Privileged Identity Management (PIM) — eligible vs active roles
- Service account governance and non-human identities
- Secrets management and hardcoded credential risks
- Break-glass emergency access procedures
- PAM for cloud infrastructure and multi-cloud environments

### 🏗️ Phase 4 — IAM Architecture & Design
- IAM architecture patterns: centralised, federated, hybrid
- Multi-cloud identity challenges and federation strategies
- IAM threat modeling using MITRE ATT&CK and STRIDE
- Zero Trust Architecture design (NIST SP 800-207)
- API security and OAuth scope design
- IAM for SaaS applications and shadow IT governance
- IAM maturity models and roadmap building
- IAM business case and ROI quantification

### 🏆 Phase 5 — Certifications & Portfolio
- SC-300: Microsoft Identity and Access Administrator exam prep
- SailPoint IdentityNow Professional certification prep
- GitHub portfolio with real IAM architecture documents
- LinkedIn profile and personal branding as an IAM specialist

## 🌱 My IAM Learning Philosophy

> **Concepts first, tools second.**

Rather than learning tools in isolation, I focus on understanding the underlying identity and security concepts deeply. This makes it possible to work with any IAM platform, not just the ones I have used before.

Each day's practice task is designed to apply the concept — not just read about it — through case studies, design exercises, policy drafting, and architecture thinking.
